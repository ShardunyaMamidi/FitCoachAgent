# Project context

I'm learning how to build AI agents and agentic apps. Treat this project as a learning exercise, not just a delivery task.

## How to work with me here

When you write or change code, don't just hand me the result — teach as you go:

- **What** the code does, in plain terms.
- **Why** you chose this approach: the reasoning behind the design.
- **What else you considered**: alternative approaches and the tradeoffs that led you to reject them.
- Call out any concepts or patterns that are specific to building AI agents (tool-calling loops, prompt structure, context management, evals, etc.) so I can build a mental model.

Favor clarity and learning over terseness. It's fine to be a bit more verbose than usual in explanations, as long as the code itself stays clean.
